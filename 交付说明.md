# StyleMind v2 — 项目交付说明

> 交付日期：2026-05-14
> 版本：v2（含大纲上传模式 + AI 智能排版 + PDF 导出）

---

## 一、项目是什么

**StyleMind** 是一个 **AI 对话式可编辑 PPTX 生成工具**，支持：
- 通过对话/上传 Markdown 大纲 → 页面 Skill 拆页 → SlideLayoutSpec 结构化排版 → 原生 PPTX 文本框、形状、图片、视频链接
- PNG/PDF 保留为预览和兜底；image-2 只做素材图，不负责整页排版和文字

## 二、快速启动

### 1. 安装依赖

```bash
cd stylemind_v2
pip install -r requirements.txt
```

主要依赖：`flask flask-cors requests openai httpx python-pptx pdfplumber pymupdf PyPDF2 pillow numpy pydantic rich opencv-python-headless`

### 2. 配置 API

编辑 `config.json`，填入你的 API 信息：

```json
{
  "api_base_url": "https://你的API地址/v1",
  "api_key": "<your_api_key>",
  "chat_model": "gpt-4o",
  "image_model": "gpt-image-2",
  "chat_api_base_url": "https://你的对话API地址/v1",
  "chat_api_key": "<your_chat_api_key>",
  "image_api_base_url": "https://你的图片API地址/v1",
  "image_api_key": "<your_image_api_key>"
}
```

> `image_model` 只用于素材图生成、补图或旧预览链路；主交付 PPTX 不把整页 PNG 塞进幻灯片。

### 3. 启动服务

```bash
cd web_ui
python3 api_server.py
```

启动后访问：**http://localhost:8080**

## 四、核心功能说明

### 4.1 大纲上传模式 (`/upload`)

**支持的文件格式：**
- `.json` — 已拆分好的结构化大纲，前端直接解析展示
- `.md` — Markdown 格式，系统自动处理：

  - **本地解析层**（零 API 调用）：
    - 识别 `Sxx｜` / `## ` 标题格式自动拆页
    - 提取子字段：标题 / 核心结论 / 页面内容 / 版式建议 / 视觉元素
    - 自动过滤非 PPT 页面章节（设计要求、排版规则等）

  - **AI 智能排版分析层**：
    - 拆页完成后调 AI 分析每页最优 layout（满版图片/左图右文/三列卡片/步骤流程/时间轴/表格）
    - 不改写内容，只做排版建议

**大纲 JSON 格式示例：**
```json
{
  "title": "PPT总标题",
  "pages": [
    {
      "index": 1,
      "title": "封面标题",
      "type": "cover",
      "layout": "纯视觉页",
      "brief": "产品大图+深色背景",
      "content": "正文内容..."
    }
  ]
}
```

### 4.2 图片生成

- 支持风格设置：品牌名称 / 色系选择 / 风格描述 / 参考图片
- SSE 流式进度返回，实时显示每张图片
- 图片直接显示在对应分页卡片下方

### 4.3 导出功能

| 功能 | 接口 | 说明 |
|------|------|------|
| **导出 PDF** | POST `/api/download-pdf` | 所有生成的图片合成一个 PDF |
| **下载 ZIP** | POST `/api/download-images-zip` | 打包所有图片为 ZIP |

### 4.4 数据持久化

- 大纲数据持久化到磁盘 `data/outline_sessions/{session_id}.json`
- 服务重启后数据不丢失
- 前端 localStorage 辅助保存状态（切页不丢）
- 历史记录面板：可恢复 / 删除历史大纲

## 五、新增的 API 接口

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/upload-outline` | POST | 上传 .md/.json 大纲，返回 session_id |
| `/api/export-pptx-structured` | POST | 根据 outline 生成原生可编辑 PPTX |
| `/api/generate-from-outline` | POST | 根据 outline 生成 PNG 预览（SSE 流式，可选兜底） |
| `/api/download-pdf` | POST | 图片合成 PDF 下载 |
| `/api/outlines` | GET | 列出所有历史上传的大纲 |
| `/api/outline/<id>` | GET | 获取某个大纲详情（含图片） |
| `/api/outline/<id>` | DELETE | 删除某个大纲 |

原有接口（`/api/chat-stream`, `/api/generate-images`, `/api/download-images-zip`, `/api/import` 等）全部保留不变。

## 七、注意事项

1. **config.json 必须配置** — 否则 API 调用会失败
2. **端口默认 8080** — 可在 `api_server.py` 底部修改
3. **data/ 目录首次运行时自动创建** — 用于存放大纲会话数据
4. **图片模型只服务素材/预览** — 当前配置的是 `gpt-image-2`，不得作为整页排版和最终 PPTX 交付底层
5. **Markdown 解析优先级**：先尝试本地正则拆页 → 不行再调 AI 提取
6. **前端 mock 回复已移除** — 之前 catch 里返回假文字的 bug 已修复
