# StyleMind v2 产品使用说明

## 一、产品简介

StyleMind v2 是一款 **AI 驱动的可编辑 PPTX 生成工具**。当前主链路是：

**大纲页面数据 → 飞博页面 Skill / SlideLayoutSpec → 原生 PPTX 文本框、形状、图片、视频链接**

PNG/PDF 仍然保留，但定位为页面预览和兜底交付，不作为最终可编辑 PPT 的底层方案。

支持两种使用模式：

- **对话模式**：用自然语言告诉 AI 你想要什么，AI 自动生成大纲并逐页出图
- **大纲上传模式**：上传已写好的 Markdown/JSON 大纲文件，自动分页后优先导出可编辑 PPTX，也可生成 PNG/PDF 预览

**技术架构**：纯本地部署（Flask 后端 + HTML 前端），数据不上传第三方，所有 AI 调用通过配置的 API Key 完成。

## 二、快速开始

### 2.1 启动服务

```bash
cd stylemind_v2/web_ui
python3 api_server.py
```

启动成功后：
- 本机访问：`http://localhost:8080`
- 局域网访问：`http://<本机IP>:8080`（同 WiFi 下的同事可直接访问）

### 2.2 首次配置

1. 打开页面后，点击左侧导航栏底部的 **⚙️ 设置**
2. 配置三套 API（对话模型 / 图片生成模型 / 嵌入模型）
3. 推荐配置：
   - 对话模型：`gpt-4o`
   - 图片生成模型：`gpt-image-2`（仅用于素材图生成/补图，不负责整页排版和文字）
   - API 地址：填写你的 OpenAI-compatible API Base URL
4. 点击 **测试连接** 确认可用，然后 **保存设置**

> 💡 如果三套 API 用的是同一个服务商，填好对话模型的地址和 Key 后，点 **复制到全部** 即可。

## 三、功能详解

### 3.1 模式一：对话式生成（主界面）

#### 步骤一：输入需求

在左侧对话框中输入你的 PPT 需求，例如：

```
帮我做一个 IMBODY 力量魔方的种草方案 PPT，
目标用户是 25-35 岁的健身人群，
风格要年轻活力，偏小红书调性
```

按 **Enter** 发送，AI 会：
1. 自动识别你的需求关键词
2. 结合知识库（如有上传参考 PPT）进行 RAG 增强
3. 流式生成结构化大纲（逐章节输出）

#### 步骤二：查看与调整大纲

生成完成后，右侧面板会显示完整的大纲结构：
- 每页显示页码、标题、类型标签、内容摘要
- 可在对话框中继续对话调整（如"第3页改成强调数据对比"）

#### 步骤三：设置风格并生成图片

点击 **🎨 根据大纲生成图片**，弹出风格设置框：

| 设置项 | 说明 | 示例 |
|--------|------|------|
| 品牌名称 | 用于 AI 识别品牌调性 | `IMBODY` / `UNITREE` |
| 品牌色系 | 下拉选择或留空自动识别 | `橙粉系` / `蓝绿系` |
| 风格描述 | 描述想要的视觉风格 | `科技感、年轻活力、小红书种草风` |
| 参考图片 | 上传一张风格参考图 | 上传喜欢的海报/PPT截图 |
| 产品参考图 | 上传实际产品照片 | 上传 PM UP 产品图 |

> 📦 **产品参考图功能**：上传产品图后，系统会智能判断哪些页面需要展示产品（如产品介绍页、使用场景页），只在需要的页面传入产品图作为视觉参考。不需要产品的页面（如时间轴、数据分析）会自动跳过，既保证效果又提升速度。
>
> ⚠️ 如果带产品图的请求超时，系统会**自动降级**为无参考图模式继续生成，不会中断。

#### 步骤四：预览与导出

- **图片画廊**：点击 **所有图片** Tab 以缩略图网格浏览全部生成的图片
- **单页修改**：对不满意的图片，点击 **✏️ 修改此页**，输入修改指令（如"背景换成深蓝渐变""标题放大"），AI 会重新生成该页
- **导出 PPT**：点击 **📊 转换为PPT**，AI 将图片转换为可编辑的 .pptx 文件（背景重绘+文字/元素分层）

### 3.2 模式二：大纲上传（推荐用于已有大纲的场景）

点击左侧导航 **📤 大纲上传** 进入独立页面。

#### 步骤一：上传大纲文件

支持两种格式：

**格式 A：JSON 结构化大纲**

```json
{
  "title": "IMBODY 种草方案",
  "pages": [
    {
      "title": "封面",
      "type": "cover",
      "layout": "满版图片-全屏背景",
      "content": "IMBODY 力量魔方 —— 你的随身私教"
    },
    {
      "title": "产品核心卖点",
      "type": "content",
      "layout": "模块化-三列卡片",
      "content": "卖点1：轻量化设计\n卖点2：AI 纠姿\n卖点3：游戏化互动"
    }
  ]
}
```

**格式 B：Markdown 大纲（推荐）**

系统支持智能解析以下 Markdown 格式：

```markdown
S01｜封面

IMBODY 力量魔方
—— 小体积·大力量 · 游戏化健身新物种


S02｜市场洞察

【核心结论】
中国居家健身市场规模突破 3000 亿，但传统器械"大、重、丑"劝退年轻人。

【页面内容】
- 90 后/00 后成为主力消费群体
- "小而美""高颜值"成为选购关键词
- 传统哑铃/杠铃闲置率高达 67%

【版式建议】满版图片-左图右文
【视觉元素】左侧数据图表，右侧年轻人在家健身场景
```

**Markdown 解析规则**：

- 支持 `Sxx｜标题`、`## 标题`、`数字.数字｜标题` 三种分隔格式
- 自动提取子字段：标题、核心结论、页面内容、版式建议、视觉元素
- 自动过滤非页面内容（设计要求、排版规则、前言附录等）
- 解析完成后 AI 会优化每页的排版布局建议
- 如果正则无法解析（非标准格式），会调用 AI 辅助提取

#### 步骤二：预览分页结果

上传后页面显示：
- **统计条**：总页数、标题
- **分页卡片列表**：每张卡片可展开查看详情（标题、内容、布局类型、排版说明）
- 支持展开全部 / 收起全部

#### 步骤三：设置风格

与主界面相同的风格设置面板：

- 品牌名称 / 品牌色系 / 风格描述
- 参考图片（风格参考）
- **产品参考图**（实际产品照片，PNG/JPG/PDF 均可）

#### 步骤四：生成图片

点击 **🎨 开始生成图片**：

- SSE 流式生成，实时显示进度（第 X/Y 页）
- 每张卡片生成完毕后会显示图片缩略图
- 生成过程中可随时 **⏹ 终止**
- 日志输出示例：
  ```
  [INFO] 第1页「封面」需要产品图 → 使用 /v1/images/edits
  [INFO] 第3页「数据分析」不需要产品图 → 标准生成
  [INFO] 第6页「产品特点」需要产品图 → 使用 /v1/images/edits
  ```

#### 步骤五：单页编辑（可选）

每张已生成的图片下方有 **✏️ 修改此页** 按钮：

1. 点击打开编辑弹窗
2. 查看当前图片
3. 输入修改指令，例如：
   - "把背景换成日落海滩"
   - "标题字体再大一点"
   - "人物换成男性"
4. AI 分析指令 → 重新生成该页图片 → 自动替换

#### 步骤六：导出

| 导出方式 | 说明 |
|----------|------|
| **📄 导出 PDF** | 所有图片合成为一个 PDF 文件下载 |
| **📦 下载 ZIP** | 所有图片打包为 ZIP（按页码命名） |

### 3.3 知识库管理

**用途**：上传已有的优秀 PPT/PDF 文件，让 AI 学习其设计风格，后续生成时自动匹配类似风格。

**操作路径**：主界面 → 左侧导航 **📚 知识库管理**

**支持格式**：`.pptx`、`.pdf`

**导入流程**：
1. 拖拽或选择文件上传
2. 点击 **导入知识库**
3. 系统自动处理：
   - PDF 拆分（超过 40MB 或 50 页时自动拆分）
   - 逐页 OCR + 布局分析
   - LLM 批量分析设计特征（每批 5 页）
   - 向量化存入知识库
4. 之后在对话模式下，AI 会自动检索相关知识库内容增强生成质量

---

### 3.4 数据持久化

系统有多层数据保护机制：

| 层级 | 说明 |
|------|------|
| **服务端文件存储** | 大纲会话保存在 `data/outline_sessions/` 目录，重启不丢失 |
| **服务端内存缓存** | 当前活跃会话常驻内存，读写更快 |
| **浏览器 localStorage** | 前端状态（大纲、图片 URL、风格设置）本地保存 |
| **历史记录** | 大纲上传页面顶部可查看/恢复/删除历史会话 |

## 四、支持的排版布局（7 种）

| 布局名称 | 适用场景 | 效果描述 |
|----------|----------|----------|
| **满版图片-全屏背景** | 封面、总结、情感共鸣页 | 全屏摄影背景 + 白字叠加 + 品牌色标签 |
| **满版图片-左图右文** | 内容对比、玩法说明、事件介绍 | 左侧摄影图 + 右侧白底文字区 |
| **模块化-三列卡片** | 创意亮点、分类展示、卖点呈现 | 三等宽圆角卡片横向排列 |
| **模块化-步骤流程** | 执行计划、动作拆解、操作指南 | STEP 编号标记 + 箭头连接 |
| **模块化-时间轴** | 策略总览、阶段规划、发展历程 | 横向时间轴 + 阶段节点详情 |
| **模块化-表格** | 数据对比、竞品分析、传播总览 | 多列表格 + 表头着色 + 斑马纹行 |
| **纯视觉页** | KV 展示、素材展示 | 全屏大图，极少文字 |

> 布局可以由 Markdown 大纲中的 `版式建议` 字段指定，也可以由 AI 根据内容自动判断。

## 五、产品参考图功能详解

### 5.1 工作原理

当你在风格设置中上传了**产品参考图**后，系统会在每页生成前自动判断：

```
当前页面内容 ──→ _page_needs_product_image() ──→ 是否需要产品图？
                                                    │
                              ┌─────────────────────┼─────────────────────┐
                              ▼                     ▼                     │
                          需要                    不需要                  │
                              │                     │                     │
                    使用 /v1/images/edits    标准 images.generate       │
                    (multipart/form-data)     (快速，不带产品图)         │
                    附带产品图二进制数据                                │
                              │                                           │
                        超时/报错？                                      │
                       ╱         ╲                                     │
                      ▼           ▼                                     │
                   降级为        返回正常                               │
                无参考图模式                                            │
```

### 5.2 智能判断规则

**以下情况会附带产品参考图**：
- ✅ 产品展示 / 介绍 / 特点 / 功能页
- ✅ 使用场景 / 应用场景页
- ✅ 含产品名称（imbody / pm up / unitree 等）
- ✅ 封面页（且内容涉及产品发布/展示）
- ✅ KV 纯视觉页

**以下情况不会附带产品参考图**（走快速通道）：
- ❌ 时间轴 / 阶段规划 / 排期表
- ❌ 数据分析 / 趋势报告 / 调研
- ❌ 策略定位 / 洞察发现
- ❌ TVC 脚本 / 分镜脚本
- ❌ 流程步骤 / 表格 / 预算 KPI

### 5.3 容错机制

- 产品图 > 512KB 时自动压缩到 1024px 以内
- `/v1/images/edits` 超时（10 分钟内）→ 自动降级为无参考图模式
- `/v1/images/edits` 返回错误 → 同样降级
- 降级后仍能正常生成图片（只是不带产品参考）

## 六、常见问题

### Q1：服务打不开 / 上传没反应

服务可能被终止了。重新启动：
```bash
cd stylemind_v2/web_ui
python3 api_server.py
```

### Q2：同事连不上我的电脑

确保：
1. 两台设备在**同一个 WiFi** 下
2. 访问地址是 `http://<你的IP>:8080`（不是 localhost）
3. 查看本机 IP：终端运行 `ipconfig getifaddr en0`
4. macOS 防火墙已关闭（系统设置 → 网络 → 防火墙 → 关闭）

### Q3：生成长时间超时

- 图片生成（特别是带产品参考图）需要较长时间，请耐心等待
- 单页最长等待 10 分钟
- 超时会自动降级重试，不会丢失进度
- 建议：57 页的大纲，首次生成预计 20-40 分钟

### Q4：生成的产品和实际产品不一样

1. 确认已在风格设置中上传了**产品参考图**
2. 查看服务日志确认是否有 `[INFO] 使用 /v1/images/edits 接口` 的记录
3. 如果某页确实不需要产品图（如数据分析页），这是正常的智能判断行为
4. 可以通过 **✏️ 修改此页** 功能单独调整不满意的产品图页面

### Q5：大纲格式不对怎么办

- **标准格式优先**：使用 `Sxx｜标题` 或 `## 标题` 作为页分隔符
- **JSON 最可靠**：如果 Markdown 解析结果不理想，改用 JSON 格式上传
- **AI 兜底**：非标准格式会调用 AI 辅助解析，但不保证 100% 准确

### Q6：如何更换 API 提供商

进入 **⚙️ 设置** 页面，分别修改三套 API 的地址和 Key：
- 对话模型 API（用于大纲生成、对话）
- 图片生成模型 API（用于出图）
- 嵌入模型 API（用于知识库向量化）

三者可以使用不同的服务商。

## 七、API 接口速查

| 方法 | 路径 | 用途 |
|------|------|------|
| POST | `/api/chat-stream` | SSE 流式对话（大纲生成） |
| POST | `/api/generate-images` | 从对话大纲生成图片（SSE） |
| POST | `/api/export-pptx-structured` | 从大纲直接生成原生可编辑 PPTX |
| POST | `/api/generate-from-outline` | 从上传大纲生成 PNG 预览（SSE，可选兜底） |
| POST | `/api/edit-image` | 单页 AI 编辑重新生成（SSE） |
| POST | `/api/upload-outline` | 上传并解析大纲文件 |
| POST | `/api/upload-product-image` | 上传产品参考图 |
| GET | `/api/outlines` | 获取大纲历史列表 |
| GET | `/api/outline/<id>` | 获取单个大纲详情 |
| DELETE | `/api/outline/<id>` | 删除大纲会话 |
| POST | `/api/download-pdf` | 导出 PDF |
| POST | `/api/download-images-zip` | 导出 ZIP 图片包 |
| POST | `/api/convert-ppt-v2` | 从整页图片反推分层 PPTX（旧兼容链路） |
| POST | `/api/import` | 导入知识库（SSE） |
| GET/POST | `/api/settings` | 获取/保存 API 配置 |
